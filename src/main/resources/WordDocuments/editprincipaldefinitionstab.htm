Edit Principal Definitions tab

This tab allows the user to assign one or more security definitions to the principal. Using this tab, the user may assign an exception to a rule that would otherwise apply to the principal.

The display above shows the tab in create mode. In edit mode, this tab presents a display-only set of fields on the left, editable fields on the right in which the user may enter changes, and a delete button at the bottom.

Edit Principal Definitions tab definition

Title

Description

Definition Name

Required. A security definition to be applied to this principal. Use the lookup  to select an appropriate definition.

Constraint Code

Required. Indicates whether access/use is allowed or denied. Click the appropriate button.

Operator

Required. Specifies the range operator for attribute values for which access or use is allowed or denied. Select the appropriate operator from the list.

Attribute Value

Required. Identifies the specific attribute value (e.g., a specific account code) to which the operator applies.

Override Deny

Check the box to allow access that is otherwise denied.

Active Indicator

 

Indicates whether this definition is active or inactive for the principal. Remove the check mark to deactivate the definition.

add

Click the add button to associate this definition with the principal.

delete

Click the delete button to dissociate this definition from the principal.
